Intro

This tutorial will show you how to make websites from scratch. It is very basic nothing fancy here. We start with html and progress towards CSS and JavaScript learning only what we use. Were going to use only code to build the website. Happy coding.

What is html? Html stands for hypertext markup language. Its the language that tells the web browser how to layout a website. Like where to place text or images.

What is CSS? CSS stands for cascade style sheets. It provides a way to paint a website to any color, size or shape. It's not really a language but more a giant collection of settings we can use to change the look and feel of a website.

What is JavaScript? JavaScript is a programming language that is used to add functionality to a website and make it interactive. It adds functionality to buttons, mouse movements, mouse clicks and much more.

Part 1: Getting Started

To start you'll need a IDE or a text editor. I prefer to use vs code. Heres a link to download it here. Or use your prefered text editor. This tutorial uses vs code. Lets start by creating a new file and calling it "main.html". Were going to start with the code explaining and examining it piece by piece, similar to putting together a lego set. This first piece of code tells html what type of document this is. Add this to the main file you just created, type at the top:

<!DOCTYPE html>

Next piece of code is a comment. Comments are ignored by html and they serve a purpose for the programmer. The purpose is to explain code and keep it organized.

<!-- This is a comment -->

Next piece of code is called a tag. Tags serve the purpose of marking out the begining and ending of the html code. They also do other things like set properties. The html tag is the main tag and all other tags are enclosed within it. The lang="en" tells the browser what language to use in our case "en" stands for english.

<html lang="en">

Next piece of code is a head tag. It holds the settings for the webpage.

<head>

This tag is called meta data. This tells html what character set we are using. UTF-8 is the character set, it is useful for translating the webpage into other languages.

<meta charset="UTF-8">

This tells html how to view the webpage on different devices. Like on a phone, tablet or a pc. The width=device-width says set the width of the page to the width of the device screen. The initial-scale=1.0 tells html to set the zoom level when the page is first loaded to 1.

<meta name="viewport" content="width=device-width,initial-scale=1.0">

This code tells the browser to render the page in the latest standard. Its specific to internet explorer.

<meta http-equiv="X-UA-Compatible" content="ie-edge">

This code sets the title for the webpage

<title>My webpage</title>

This code is a closing tag for head

</head>

Now the body tag, this is where we put most of our code

<body>

Then the closing body tag

</body>

Then the closing html tag

</html>

Now for the complete boiler plate html code. This code is what each html file must have in order to function correctly.

<!DOCTYPE html>
<!-- This is a comment -->
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width,initial-scale=1.0">

I'll explain the CSS code now line by line. .header is a css class. It could be any name you choose .anyNameHere then inside curley brackets { css settings here}. There's many many CSS settings to choose from. So I use a reference to find the setting I want and an explanation. The reference I prefer to use is https://cssreference.io/.

.header{
/* Settings here */
}

This sets the width to 100%. 100% means it fills the screen from one end to the other. It can also be set to pixels like so 100px this means the width will be 100 pixels. I have it set to 100% because I want the header to be from one end of the window to the other end of the window even when the window is resized. Remember to add a semi colon to the end of each setting.

width:100%;

This sets the height to 150 pixels. It could be set to % or vh or auto. Try it out for yourself.

height:150px;

This sets the background color to a hex value. #ccc is light gray in hexadecimal.

background-color:#ccc;

border is a property that wraps a border around an element. The 1px means one pixel for the border-width. solid means the border line is continuous and black is the border-color.

border:1px solid black;

padding adds space inside an html element. So in our example, the title First WebPage will get 10 pixels of extra space on all four sides from the border edge.

padding:10px;

margin adds space outside an html element. So in our example, First WebPage at the very top and the second First WebPage just below it will have a 20 pixel space added in between. 20px will add a 20 pixel space to the top and bottom and auto will handle the left and right by centering the element horizontally.

margin:20px auto;

box-sizing is a property that specifies whether to include the border and padding in the width and height of the element. In our example we set it to border-box, that means the padding and border will be included in the total width and height. If we set it to content-box the total width and height will be larger since the padding and border are added to the total.

box-sizing:border-box;

overflow decides how the content will be displayed when it becomes larger than its container. When the content overflows in our example a scroll bar will be displayed since the setting is auto.

overflow:auto;

If you find yourself wondering why didn't a setting change the way it's supposed to. In firefox right click anywhere on your webpage and select Inspect(Q) or press f12 make sure the firefox window is in focus. Play around with it and get used to using it, its very useful. We will be using Inspector and Style Editor for now.

I'll explain the button class found in "button.js". When an instance of the button class is made it calls the class constructor and passes the "id" into the constructor function. "id" is a string type and it holds the unique html element id. It is then passed into the function "document.getElementById(id)" this function returns the html element and stores a reference in "this._button". Then we check if "this._button" is null and if it is send a message to the console. An event listener is added to the button, when the button is clicked with the mouse "toCall()" is executed. And some class members are set.

class button{ // encapsulates all functionality of a button
    // constructor is called when a new button is made
    constructor(id)
    {
        // returns the html element that matches the id
        this._button = document.getElementById(id);
        
        // sends a message to the console, remember when you press f12 in the browser and the debug window comes up
        // thats where the message is sent to
        if(this._button==null){console.log("error cant get element id");}
        
        // adds an event listener
        // when this._button is clicked it will call the class function toCall()
        this._button.addEventListener("click",this.toCall.bind(this));
        
        // class members
        this.ID = id;
        this._pressed = false;
        this.PressCount =0;
    }
    // rest of the class functions
}

In order to add an event listener to a class member function you must pass in "this.toCall.bind(this)". "this" in javascript depends on how the method is called, for instance if it is called from an event handler "this" would refer to the element the event was acted upon-in this case, "this._button". That is why ".bind(this)" is used, it creates a new function with the same body and scope as this.toCall, but with "this" permanently bound to the value provided, which is an instance of the class. This ensures that inside "this.toCall", "this" refers to the class instance, not the DOM element. It allows you to safely use "this" within "toCall" to refer to class properties and methods.

The rest of the class is pretty self explanitory. Just some basic functions.

class button{
    // constructor as before
    
    // call to release the button
    Release(){
        this._pressed = false; 
        this.ResetPressCount();
    }
    
    // called when a button is clicked
    Press(){
        this._pressed = true; 
        this.PressCount++;
    }
    
    // useful for checking if the button was pressed
    IsPressed() {
        return this._pressed;
    }
    
    // function that is called when a button is clicked
    toCall(Event){
        this.Press(); 
        ButtonLogic(Event);
    }
    
    // PressCount reset
    ResetPressCount(){
        this.PressCount=0;
    }
    
    // change the title of the button
    ChangeButtonText(_text){
        this._button.innerHTML = _text;
    }

    // returns the html button element
    GetButtonElem(){
        return this._button;
    }

    // returns the string id
    GetId(){
        return this.ID;
    }
}

I will explain "main_script.js" with comments below. 

// make an instance of a button 
// const, since the button does not change
// pass in "b1" the id of the html element
const Mybutton = new button('b1');

// get the title html element
let title = document.getElementById('t1');

// global function that toCall() in the button class calls
function ButtonLogic(Event){
    // check if the button is pressed, if it is change the title to "Hello World"
    if(Mybutton.IsPressed()){
        title.textContent = 'Hello World';

        // oops forgot to mention a call to release the button
        // this is important when there is more than one button on the page
        // because all buttons have toCall() which calls the same ButtonLogic() function
        MyButton.Release();
    }
}
